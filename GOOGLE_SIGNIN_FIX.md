# Fixing Google Sign-In 400 Error

## The Problem

You're getting a 400 "malformed request" error because when using `expo-auth-session` with `useProxy: true`, you **must use the Web OAuth client ID**, not the iOS/Android client IDs.

## The Solution

### Step 1: Use Web Client ID

When using the Expo auth proxy, Google expects a **Web OAuth client ID**, not platform-specific ones.

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Navigate to **APIs & Services** → **Credentials**
3. Create or find your **Web application** OAuth client
4. Copy the **Client ID** → this is your `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID`

### Step 2: Add Redirect URI to Web Client

The redirect URI generated by Expo's proxy needs to be added to your **Web OAuth client** (not iOS/Android):

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Navigate to **APIs & Services** → **Credentials**
3. Find your **Web application** OAuth client (the one you're using for `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID`)
4. Click on the client name to **edit** it
5. Scroll down to **Authorized redirect URIs**
6. Click **+ ADD URI** button
7. **Find your exact redirect URI first** (Google doesn't accept wildcards):

   **Method 1: Check the error message**

   - When you try to sign in and get the 400 error, check the error details
   - It might show the redirect URI that was attempted
   - Look for a URL like `https://auth.expo.io/@something/Saute`

   **Method 2: Log it in your app (Recommended)**

   - Add this temporarily to your login screen (`app/(auth)/login.tsx`):

     ```typescript
     import * as AuthSession from "expo-auth-session";

     // Add this in your component, maybe in a useEffect:
     useEffect(() => {
       const redirectUri = AuthSession.makeRedirectUri();
       console.log("🔗 Redirect URI:", redirectUri);
       Alert.alert("Redirect URI", redirectUri); // Or just check console
     }, []);
     ```

   - Run your app and check the console/logs or the alert
   - Copy the exact URI that's printed

   **Method 3: Check your Expo account**

   - Run: `npx expo whoami` to see if you're logged in
   - If logged in, your redirect URI will be: `https://auth.expo.io/@[your-username]/Saute`
   - If not logged in, it might use a default format

8. **Add the exact redirect URI you found:**

   - Click **+ ADD URI**
   - Paste the exact URI (e.g., `https://auth.expo.io/@your-username/Saute`)
   - Click **+ ADD URI** again
   - Add: `http://localhost:19006` (for local development)

9. After adding each URI, click **SAVE** at the bottom of the page

**Important Notes:**

- Google OAuth Web clients **only accept `http://` or `https://` URIs**
- The `exp://` protocol is **not supported** here (it's only for Expo's internal routing)
- You can add multiple redirect URIs - just click **+ ADD URI** for each one

### Step 3: Update Your .env File

Make sure you have the **Web client ID** in your `.env`:

```bash
# Use WEB client ID for expo-auth-session with proxy
EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID=your_web_client_id_here

# iOS and Android client IDs are still needed for native builds
EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID=your_ios_client_id_here
EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID=your_android_client_id_here
```

### Step 4: Restart Your App

After updating the `.env` file:

```bash
# Stop your dev server (Ctrl+C)
# Then restart with cleared cache
npx expo start --clear
```

## Why This Happens

- **Expo's auth proxy** creates a web-based redirect URI (`https://auth.expo.io/...`)
- Google sees this as a **web redirect**, so it requires a **Web OAuth client**
- Using iOS/Android client IDs with a web redirect causes the 400 error

## Alternative: Use Native Client IDs (Without Proxy)

If you want to use native client IDs instead, you'd need to:

1. **Not use the proxy** (`useProxy: false`)
2. Configure **custom URL schemes** in your app
3. Handle the redirect manually

But using the Web client ID with proxy is **much easier** and recommended for Expo apps.

## Summary

✅ **Use Web OAuth client ID** when using `expo-auth-session` with `useProxy: true`  
✅ **Add redirect URI** to Web client's authorized redirect URIs  
✅ **Restart app** after updating `.env`

The code has been updated to use the Web client ID automatically. Just make sure your `.env` has `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID` set correctly!
